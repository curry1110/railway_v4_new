## 🔧 核心修復：徽章重複更新

### 問題背景
原有邏輯存在潛在的競態條件：
```javascript
// 舊邏輯（不安全）
SELECT ... // 檢查是否存在
IF NOT EXISTS:
  INSERT ...
ELSE:
  UPDATE ...
```

### 解決方案
使用 MySQL 的 **原子性操作**：
```sql
INSERT INTO user_stamps (...) VALUES (...)
ON DUPLICATE KEY UPDATE visit_count = visit_count + 1
```

### 改進效果
✅ **一次性操作**：消除競態條件  
✅ **精確判斷**：根據 `affectedRows` 判斷是否首次解鎖  
✅ **造訪追蹤**：返回 `visit_count` 用於前端提示  
✅ **事務安全**：在事務內完成所有操作  

### 前端提示邏輯
- **首次解鎖**：`🎉 獲得新印章：[站名]！`
- **重複造訪**：`再次造訪 [站名]！目前造訪 N 次`

## 🔐 安全性考量

1. **資料庫一致性**：使用 `ON DUPLICATE KEY UPDATE` 確保原子性
2. **事務管理**：所有操作在事務內完成，失敗時自動回滾
3. **輸入驗證**：站點代碼驗證，防止無效請求
4. **SQL 注入防護**：使用參數化查詢
